# 淘宝网购 UI 自动化测试 Skill

## 一、这个 Skill 的用途

当用户通过飞书或 OpenClaw 发送淘宝自动化测试任务时，使用本 Skill。
本 Skill 必须使用 `taobao-cdp` 浏览器 Profile。

该 Profile 必须连接到一个固定的 Chrome 用户数据目录：

```text
~/.openclaw/browser-profiles/taobao
本 Skill 必须使用 `taobao-cdp` 浏览器 Profile。

```
适用场景包括：

- 执行淘宝 UI 自动化测试
- 执行网购 Agent 测试
- 搜索淘宝商品并加入购物车
- 根据好评率筛选商品
- 将淘宝测试结果回传飞书

用户消息中如果出现以下关键词，应优先考虑使用本 Skill：

- 淘宝UI测试
- 淘宝自动化测试
- 网购Agent测试
- 执行淘宝购物车测试
- taobao shopping test
- 加入购物车测试

---

## 二、任务输入格式

用户推荐使用 JSON 格式下发任务。

标准任务格式如下：

```json
{
  "task_id": "TB_TEST_001",
  "keyword": "无线鼠标",
  "min_good_rate": 99,
  "target_count": 1,
  "action": "add_to_cart",
  "max_scan": 10
}
```
字段说明：

task_id：任务编号，用于区分不同测试任务。
keyword：搜索关键词，例如“无线鼠标”“蓝牙耳机”“机械键盘”。
min_good_rate：最低好评率阈值，例如 99 表示好评率需要大于 99%。
target_count：需要加入购物车的商品数量，默认 1 个。
action：执行动作，目前只允许 add_to_cart，也就是加入购物车。
max_scan：最多扫描多少个候选商品，默认 10 个。

如果用户没有提供完整字段，使用以下默认值：

min_good_rate 默认值：99
target_count 默认值：1
action 默认值：add_to_cart
max_scan 默认值：10

##三、安全原则
只能使用已经登录好的淘宝测试账号。
不能向用户索要淘宝账号、密码、短信验证码、支付密码。
不能保存淘宝账号、密码、Cookie、Token、支付信息。
不能绕过验证码、滑块验证、短信验证、人脸验证或淘宝风控。
如果页面出现验证码、安全验证、短信验证、风险提示，必须立即停止。
只允许点击“加入购物车”。
禁止点击“立即购买”。
禁止点击“提交订单”。
禁止点击“去结算”。
禁止点击“付款”或任何支付相关按钮。
每次执行任务后，都需要返回结构化测试结果。
执行过程中尽量保存截图、当前 URL、商品信息和失败原因。
如果检测到淘宝未登录，必须停止任务，并返回：

```text
LOGIN_REQUIRED
```

如果检测到验证码、滑块验证、短信验证、人脸验证、安全验证或风控页面，必须停止任务，并返回：

```text
CAPTCHA_REQUIRED
```

或：

```text
RISK_CONTROL
```

## 四、完整执行流程
### 第一步：解析任务

从用户消息中提取任务参数。

需要识别以下字段：

- task_id
- keyword
- min_good_rate
- target_count
- action
- auth_mode
- max_scan

如果用户没有提供 JSON，但使用自然语言描述了任务，也应尽量提取关键词。

例如：

```text
帮我在淘宝搜索无线鼠标，找好评率大于99%的商品，加入购物车。
```

应解析为：

```json
{
  "keyword": "无线鼠标",
  "min_good_rate": 99,
  "target_count": 1,
  "action": "add_to_cart",
  "auth_mode": "existing_session",
  "max_scan": 10
}
```
如果无法识别搜索关键词，返回：

```text
INVALID_TASK
```
---

### 第二步：打开谷歌浏览器
如果检测到谷歌浏览器打开，则跳过这一步

##第三步：在谷歌浏览器内打开淘宝网页
使用浏览器打开：


```text
https://www.taobao.com
```

等待页面加载完成。

需要检查以下内容：

1. 页面是否成功打开。
2. 页面是否为空白。
3. 页面是否加载失败。
4. 是否跳转到登录页。
5. 是否出现验证码页面。
6. 是否出现安全验证页面。
7. 是否出现风控页面。

如果页面无法打开，返回：

```text
BROWSER_ERROR
```

---
### 第三步：检查登录状态

打开淘宝首页后，检查当前是否处于登录状态。

如果页面中出现以下内容，说明可能未登录：

- 亲，请登录
- 请登录
- 登录淘宝
- 扫码登录
- 账号登录
- 手机号登录
- 免费注册
- 登录后查看

如果出现上述情况，停止自动化任务，返回：

```text
LOGIN_REQUIRED
```
并给出提示：

```text
当前淘宝测试账号未登录或登录态已过期，请在浏览器窗口中手动完成登录，然后重新下发任务或发送“继续执行”。
```
如果页面中出现以下内容，说明可能已经登录：

- 我的淘宝
- 购物车
- 收藏夹
- 已买到的宝贝
- 用户昵称
- 账号中心

如果已经登录，继续执行搜索流程。

---

### 第四步：检查验证码和风控

如果页面中出现以下内容，必须停止任务：

- 验证码
- 滑块验证
- 安全验证
- 风险验证
- 短信验证
- 人脸验证
- 扫码确认
- 异常访问
- 访问受限
- 请完成验证
- 账号存在风险
- 环境异常

如果出现验证码、滑块验证或人机验证，返回：

```text
CAPTCHA_REQUIRED
```

如果出现账号风险、安全校验或访问受限，返回：

```text
RISK_CONTROL
```

不得尝试绕过这些验证。

---

### 第五步：搜索商品

在淘宝首页定位搜索框。

优先查找以下元素：

1. 搜索输入框。
2. 占位文字包含“搜索”的输入框。
3. 名称包含 `q` 的输入框。
4. 淘宝首页主搜索框。

输入用户指定的 `keyword`。

例如：

```text
无线鼠标
```

然后执行搜索。

搜索方式可以是：

1. 点击搜索按钮。
2. 或者按 Enter 键。

搜索后等待搜索结果页加载完成。

如果没有搜索结果，返回：

```text
NO_RESULT
```

---

### 第六步：扫描搜索结果

从搜索结果页读取候选商品列表。

最多扫描 `max_scan` 个候选商品。

每个候选商品应尽量提取以下信息：

- 商品标题
- 商品价格
- 店铺名称
- 商品链接
- 销量信息
- 评价信息
- 好评率信息
- 是否广告位
- 是否可进入详情页

扫描规则：

1. 优先扫描自然搜索结果。
2. 避免误选广告占位。
3. 不要点击“立即购买”。
4. 不要点击“去结算”。
5. 不要点击任何支付相关按钮。
6. 商品链接必须指向淘宝或天猫商品详情页。
7. 如果商品卡片信息不完整，可以进入详情页继续解析。

如果搜索结果页无法读取商品，返回：

```text
NO_RESULT
```

---

### 第七步：解析好评率

本 Skill 的核心筛选条件是：

```text
商品好评率 > min_good_rate
```

例如：

```text
商品好评率 > 99%
```

好评率解析优先级如下：

第一优先级：搜索结果页直接展示好评率。

例如页面出现：

```text
好评率 99.6%
```

则直接解析。

第二优先级：打开商品详情页，在详情页查找好评率。

重点查找以下文字附近的信息：

- 好评率
- 宝贝评价
- 累计评价
- 全部评价
- 评价
- 店铺评分
- 描述相符
- 服务态度
- 物流服务

第三优先级：如果详情页仍然没有明确显示好评率，则标记该商品为：

```text
RATE_NOT_FOUND
```
### 第八步：筛选符合条件的商品

如果商品明确显示好评率，并且满足：

```text
good_rate > min_good_rate
```

则认为该商品符合条件。

例如：
```text
min_good_rate = 99
good_rate = 99.6
```

该商品符合条件。
```json
{
  "title": "某品牌无线鼠标",
  "price": "59.90",
  "shop": "某某旗舰店",
  "good_rate": 99.6,
  "product_url": "https://item.taobao.com/xxx",
  "matched": true
}
```

如果扫描完 `max_scan` 个商品后，仍然没有符合条件的商品，返回：

```text
NO_MATCHED_PRODUCT
```

如果所有候选商品都没有找到明确好评率，返回：

```text
RATE_NOT_FOUND
```

---

### 第九步：打开商品详情页

对符合条件的商品，打开商品详情页。

进入详情页后再次检查：

1. 是否仍然是同一个商品。
2. 商品是否下架。
3. 商品是否缺货。
4. 是否出现验证码。
5. 是否出现风控。
6. 是否需要选择规格。
7. 是否存在“加入购物车”按钮。

如果商品下架或不可购买，返回：

```text
OUT_OF_STOCK
```

如果出现验证码，返回：

```text
CAPTCHA_REQUIRED
```

如果出现风控，返回：

```text
RISK_CONTROL
```

---

### 第十步：选择规格

如果商品详情页要求选择规格，例如：

- 颜色
- 尺码
- 套餐
- 版本
- 型号
- 数量
- 款式

则优先选择页面中第一个可用规格。

选择规格时必须谨慎：

1. 只能选择可点击的规格。
2. 不能选择灰色、缺货、不可用规格。
3. 不要随意修改购买数量。
4. 默认购买数量保持为 1。
5. 如果无法明确选择规格，返回：

```text
SKU_REQUIRED
```

---

### 第十一步：加入购物车

只允许点击：

```text
加入购物车
```

点击前必须确认按钮文本不是以下内容：

- 立即购买
- 马上抢
- 提交订单
- 去结算
- 结算
- 付款
- 确认支付
- 购买
- 下单

如果没有找到“加入购物车”按钮，返回：

```text
ADD_CART_BUTTON_NOT_FOUND
```

点击“加入购物车”后，等待页面反馈。

成功信号可能包括：

- 成功加入购物车
- 已加入购物车
- 加入成功
- 购物车数量增加
- 商品已成功加入购物车

如果看到成功信号，记录：

```text
ADD_CART_SUCCESS
```

如果点击后仍提示选择规格，返回：

```text
SKU_REQUIRED
```

如果点击失败，返回：

```text
ADD_CART_FAILED
```

## 七、错误码定义

统一使用以下错误码。

| 错误码 | 含义 | 处理方式 |
|---|---|---|
| SUCCESS | 任务成功完成 | 回传成功结果 |
| PARTIAL_SUCCESS | 找到商品但部分操作失败 | 回传商品信息和失败原因 |
| INVALID_TASK | 任务格式错误 | 提示用户修正任务参数 |
| LOGIN_REQUIRED | 淘宝未登录或登录态过期 | 提示用户人工登录 |
| CAPTCHA_REQUIRED | 出现验证码或滑块验证 | 停止任务，不绕过验证 |
| RISK_CONTROL | 出现风控或安全验证 | 停止任务 |
| NO_RESULT | 没有搜索结果 | 回传无结果 |
| RATE_NOT_FOUND | 无法找到明确好评率 | 回传说明，不编造数据 |
| NO_MATCHED_PRODUCT | 没有符合好评率条件的商品 | 回传扫描结果 |
| SKU_REQUIRED | 需要选择规格但无法安全选择 | 停止加购 |
| OUT_OF_STOCK | 商品缺货或下架 | 尝试下一个候选商品或返回失败 |
| ADD_CART_BUTTON_NOT_FOUND | 没有找到加入购物车按钮 | 返回失败 |
| ADD_CART_FAILED | 点击加入购物车后未成功 | 返回失败 |
| BROWSER_ERROR | 浏览器连接、页面加载或元素定位失败 | 返回失败 |
| UNKNOWN_ERROR | 未知异常 | 返回失败并记录原因 |
## 八、异常处理策略

### 1. 登录失效

如果发现淘宝未登录：

1. 停止任务。
2. 不尝试输入账号密码。
3. 不向用户索要密码。
4. 回传 `LOGIN_REQUIRED`。
5. 提示用户在浏览器中人工登录。
6. 用户登录完成后，可重新下发任务。

返回示例：

```text
当前淘宝测试账号未登录或登录态已过期。请在浏览器窗口中手动登录淘宝后，重新发送任务。
```

---

### 2. 验证码或滑块

如果出现验证码、滑块或人机验证：

1. 停止任务。
2. 不尝试破解。
3. 不尝试绕过。
4. 回传 `CAPTCHA_REQUIRED`。

返回示例：

```text
淘宝页面出现验证码或滑块验证。出于安全要求，Agent 已停止自动化操作，请用户人工处理。
```

---

### 3. 风控或安全验证

如果出现风险提示、安全验证、短信验证、人脸验证：

1. 停止任务。
2. 不处理验证码。
3. 不继续点击。
4. 回传 `RISK_CONTROL`。

返回示例：

```text
淘宝触发安全验证或风控提示，Agent 已停止执行。
```

---

### 4. 好评率不存在

如果商品页面没有明确好评率：

1. 不要编造。
2. 不要用销量代替。
3. 不要用评分代替。
4. 标记为 `RATE_NOT_FOUND`。
5. 继续扫描下一个商品。
6. 如果所有商品都没有好评率，返回 `RATE_NOT_FOUND`。

---

### 5. 加入购物车失败

如果点击“加入购物车”失败，需要判断失败原因：

可能原因包括：

- 没有选择规格。
- 商品缺货。
- 按钮不存在。
- 页面加载失败。
- 弹窗遮挡。
- 登录失效。
- 触发验证码。
- 商品下架。

根据原因返回对应错误码。

---

## 九、继续执行规则

如果任务因为 `LOGIN_REQUIRED` 暂停，用户可以人工登录淘宝后发送：

```text
继续执行 TB_TEST_001
```

收到继续执行请求后，应：

1. 根据 `task_id` 查找最近一次任务上下文。
2. 重新打开淘宝首页。
3. 再次检查登录状态。
4. 如果已登录，继续搜索和加购流程。
5. 如果仍未登录，继续返回 `LOGIN_REQUIRED`。

如果无法找到历史任务上下文，应提示用户重新下发完整任务。

## 十、飞书回传要求

通过飞书回传结果时，消息必须简洁、清晰、可追踪。

必须包含：

## 异常流量验证处理规则

如果淘宝页面出现以下内容：

- Sorry, we have detected unusual traffic from your network.
- Please slide to verify
- 滑动验证
- 异常流量
- 安全验证
- 验证码
- 访问受限

说明平台触发了异常流量检测或人机验证。

此时 Agent 必须停止自动化操作，不得尝试绕过验证。

允许的处理方式只有两种：

1. 返回 `RISK_CONTROL` 或 `CAPTCHA_REQUIRED`，并提示用户人工处理。
2. 在本地调试模式下，暂停流程，等待用户在浏览器中人工完成验证后继续。

禁止执行以下行为：

1. 禁止自动拖动滑块。
2. 禁止模拟轨迹绕过验证。
3. 禁止伪装浏览器指纹绕过检测。
4. 禁止更换代理绕过风控。
5. 禁止反复重试导致账号或 IP 风险升高。
